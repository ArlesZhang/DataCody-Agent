## Wayland + GNOME 46 + Tiling Assistant using guide

Wayland + GNOME 46 下使用 **Tiling Assistant** 来管理 multi-monitor 是非常稳妥的选择。它本质上是 GNOME 原生边缘吸附（edge-tiling）的增强版，不会像 Forge 那样强行全局自动平铺，所以在多屏 + Wayland 下冲突极少、稳定性高。

Tiling Assistant 的核心理念：**保留 GNOME 的浮动窗口为主**，只是把默认的“左右两列”扩展到**2×2 网格（或更多自定义）**，并提供 Windows 风格的 Snap Assist 弹出选择器 + 鼠标拖拽分屏 + 键盘快捷键。

### 快速入门（multi-monitor 重点）

1. **确认已启用 & 基本鼠标用法**
   - 打开任意窗口，按住标题栏拖动到屏幕**边缘**（左/右/上/下） → 自动吸附半屏。
   - 拖到**角落**（左上/右上/左下/右下） → 吸附 1/4 屏。
   - 吸附后，屏幕剩余空间会出现 **Tiling Popup**（类似 Windows Snap Assist 的小窗口列表），点击/选择其他打开的应用来填充剩余区域。
   - 多屏行为：每个显示器独立计算吸附区域 → 拖窗口到**副屏边缘**一样会触发半屏/1/4屏吸附，**不跨屏自动桥接**（这是 GNOME Wayland 的限制，不是扩展问题）。
   - 调整大小：已吸附的窗口，用鼠标拖动**边缘**即可 resize（会保持比例，避免畸形）。

2. **推荐先改的设置（Extensions 设置里打开 Tiling Assistant 的齿轮图标）**
   - **General** 页：
     - 启用 **"Enable Tiling Popup"**（默认开）。
     - 启用 **"Enable Gaps"** → 设置 8–16 px 间隙（美观 + 防误触）。
     - **"Tile Groups"**：开启 → 同组窗口一起 raise/resize（非常实用，比如把浏览器 + 终端组队）。
   - **Keybindings** 页（最重要！）：
     - 默认很多用 **Super + 数字小键盘（KP）**，但很多笔记本/紧凑键盘没有小键盘 → 强烈建议改成你习惯的组合。
     - 常用快捷键示例（推荐自定义）：

| 功能                  | 默认快捷键（示例）     | 推荐改成（无小键盘时）     | 说明 |
|-----------------------|-------------------------|-----------------------------|------|
| Tile to left          | Super + KP 4           | Super + ←                  | 左半屏 |
| Tile to right         | Super + KP 6           | Super + →                  | 右半屏 |
| Tile to top           | Super + KP 8           | Super + ↑                  | 上半屏（垂直分屏） |
| Tile to bottom        | Super + KP 2           | Super + ↓                  | 下半屏 |
| Top-left corner       | Super + KP 7           | Super + Shift + ← + ↑      | 左上 1/4 |
| Top-right corner      | Super + KP 9           | Super + Shift + → + ↑      | 右上 1/4 |
| Center window         | （默认无）             | Super + C 或 Shift + C     | 居中浮动窗口 |
| Toggle Tile Editing Mode | （默认无）          | Super + E 或 Super + T     | 进入编辑模式（下面详解） |

   - **Tile Editing Mode**（超级实用键盘模式）：
     - 进入后，用方向键选中一个 tiled 窗口的**边缘**。
     - 按 **Super + 方向键** 标记要调整的那一边。
     - 再按 **Super + 方向键** 来增大/缩小该边。
     - 适合精确调整多屏布局。

3. **Multi-Monitor 实用技巧**
   - **每个屏独立**：在主屏上做 2×2 网格，副屏保持单窗口或简单左右分屏，不会互相干扰。
   - **移动窗口到另一屏**：用 GNOME 原生快捷键 **Super + Shift + ←/→**（把窗口扔到左/右显示器）。
   - **工作区建议**：GNOME 设置 → Multitasking → Multi-Monitor → 把副屏设为“非工作区”（不跟随主屏切换 workspace），这样副屏可以固定放参考窗口（如聊天/监控工具）。
   - **常见痛点解决**：
     - 某些应用不听吸附 → 试试先最大化再拖。
     - 缩放不同（e.g. 主屏 100%、副屏 150%）→ 正常工作，但 popup 可能位置微偏（Wayland 常见，已知行为）。
     - 想更高级布局 → 可以用 **Ctrl + 拖拽** 已 tiled 窗口来分割/重组（DND 模式创建任意分屏）。

4. **日常 workflow 建议（multi-monitor）**
   - 主屏：浏览器左半 + IDE/编辑器右半（Super + ← / →）。
   - 副屏：固定终端/文档/聊天（拖到边缘吸附半屏，或全屏）。
   - 需要 4 格 → 依次拖到四个角落 → popup 会自动提示填充。
   - 全部键盘流 → 先用 Super + 方向键快速分屏，再 Super + E 进入编辑模式微调。
